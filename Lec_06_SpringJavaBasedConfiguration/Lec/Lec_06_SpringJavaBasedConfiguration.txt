* Java Based Configuration 

    За да работим с Java Based не се нуждам от xml файл, а се нуждаем от Java клас, в който са дефинирани всичките beans
    При Java конфигурацията applicationContext се създава по следния начин: ApplicationContext applicationContext = new AnnotationConfigApplicationContext(AppConfig.class), където:
    - AnnotationConfigApplicationContext е самият клас, който се използва, когато работим с Java based configuration
    - AppConfig е специален клас, написан от програмиста, в който са дефинири всички beans 

    За да посочим, че даден клас ще съдържа методи за създаване на Beans, трябва да добавим анотацията @Configuration над декларирането на класа
    # Това позволява на Spring IoC да търси този клас и чрез методите му да създаде нужните beans 

    За разлика от xml начина, тук дефинирането на Bean става чрез @Bean и метод, който връща новосъздадения обект 
    Виж AppConfigExample.java
    # Въпреки, че тук трябвва да се напише return new, Spring работи по останалите задачи, като инжектиране на обекта както и менижиранеот му
    # Ако не укажем името на Bean-a, Spring ще вземе като име за Bean от името на метода 

* @Scope 

    Както във xml конфигурацията, когато създадем Bean в IoC контейнера, той ще бъде singleton by default и трябваше да му променим scope-a

    При Java конфигурацията, когато дефинираме bean чрез @Bean под тази анотация можем да добавим и @Scope("prototype") или @Scope("singleton"), за да:
    - или да го направим singleton
    - или да го направим не синглетон, тоест да можем да създаваме обекта няколко пъти 

* @Autowired, @Primary, @Qualifier 

    @Autowired е начин чрез който Spring автоматично ще inject-не зависимостите на даден клас
    # Възможно е да станат проблеми, ако нужното ни dependency за клас, има няколко имплементации или наследници 

    @Primary ни позволява да кажем на Spring да inject един bean с приоритет, в случай ако има други подходящи bean за инжектиране 

    @Qualifier ни позволява да въведем точното има на bean-a, който желаем да бъде инжектиран в класа 

    Note: Горните изречение се използват, ако ние дефинираме всичко в Java класа за дефиниране на beans-ове, възможно е и тези анотации да се използват в самият клас
    Виж Exercise папката, за да се ориентираш
    

* @Component, @ComponentScan 

    @Component въвежда концепцията за Managed Beans, когато използваме тази анотация Spring разбира, че той е отговорен за създаването и управлението на жизнения му цикъл
    
    @ComponentScan е анотацията, чрез която казваме на Spring в кой пъкет трябва да гледа класовете
